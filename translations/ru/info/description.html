Николай берет очень необходимый ему отпуск. Он будет путешествовать по некоторым менее известным странам, каждая из которых имеет свою уникальную валюту.

При совершении покупок Николай хотел бы использовать минимально возможное количество монет. Например, в Outer Leftopia есть монеты номиналом 1, 3 и 5 шиллингов. Он хочет купить сувенир, который стоит 13 шиллингов. Он может сделать это, используя две 5-ти шиллинговых монеты и одну 3-х шиллинговую.

Исходите из того, что Николай имеет доступ к бесконечному количеству монет каждого номинала. (У него большой банковский счет и доступ к банкомату).

Входные данные: Два аргумента. Первый аргумент - целое число (int): цена покупки. Вторым является список целых чисел (ints): номиналы доступных монет.

Выходные данные: Целое число (int). Минимальное количество монет, которые Николай может использовать для совершения покупки. Если цена не может быть определена с доступными номиналами, верните None.

Пример:

{% if interpreter.slug == "js-node" %} 
makingChange(8, [1, 3, 5]) == 2
makingChange(12, [1, 4, 5]) == 3
{% else %} 
making_change(8, [1, 3, 5]) == 2
making_change(12, [1, 4, 5]) == 3
{% endif %}

Как это используется: Помимо помощи Николаю разменять деньги, это пример проблемы комбинаторной оптимизации.

Предварительное условие: Входные данные - все положительные целые числа.
